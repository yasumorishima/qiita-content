# 【完全ガイド】YomiToku v0.9.5：日本語OCRの新時代とGoogle Colabで今すぐ試せる使い方

日本語文書のデジタル化に革命をもたらしているAI-OCRエンジン「**YomiToku（読みトク）**」。2024年11月のベータ版公開から1年足らずで**GitHub 876スター**を獲得し、**20社以上の企業**が導入する注目のツールです。

本記事では、最新版v0.9.5の新機能から技術的な詳細、他OCRツールとの比較、そして**Google Colabで今すぐ試せる実践方法**まで、YomiTokuのすべてを徹底解説します。

## YomiTokuとは：日本語特化AI文書解析エンジンの全貌

### 日本語OCRの課題を根本から解決

従来の多言語対応OCRツールでは、日本語特有の課題——縦書き、7,000字以上の文字種、複雑なレイアウト、手書きとの混在——に十分対応できませんでした。

YomiTokuは、開発者の木之下滉大郎氏（MLism株式会社CEO）が、これらの課題を根本から解決するために開発した「**日本語ファースト**」のDocument AIです。

### 4つのAIモデルによる統合解析

YomiTokuの革新性は、単一の統合されたパイプラインにあります：

1. **テキスト検出モデル**：画像内の文字領域を高精度特定
2. **テキスト認識モデル**：7,000字以上の日本語文字を認識（ParseQアーキテクチャベース）
3. **レイアウト解析モデル**：段落、図、表、ヘッダー、フッターなどの文書構造を識別
4. **表構造認識モデル**：罫線のない複雑な表でも、セルの結合状態まで正確に把握

これら4つのモデルが連携することで、**単なる文字列の羅列ではなく、人間が読むのと同じように文書の意味構造を保った出力**を実現しています。

## 最新版v0.9.5：実用性をさらに高める新機能

### 読み順可視化で品質確認が容易に

2025年9月にリリースされたv0.9.5の最大の注目点は、**レイアウト解析における読み取り順番号の可視化機能**です。複雑なレイアウトの文書で、AIがどの順序でテキストを読み取っているかを視覚的に確認できるようになりました。

これにより、2段組の論文や複雑な報告書での読み取り順序の検証が簡単になり、品質管理が大幅に向上しています。

### より柔軟な開発者向け機能

- **Text Recognizerの単体呼び出し**：文字認識モデルを独立して利用可能
- **DocumentAnalyzerのパラメータ拡張**：より細かい制御が可能
- **相対パス問題の修正**：結果ファイルが隠しファイルになる問題を解決

## 革命的な手書き文字認識：v0.8.0の衝撃的進化

### 印字と手書きを単一モデルで同時認識

YomiToku最大のブレークスルーは、**v0.8.0（2025年4月）で実装された手書き文字認識機能**です。従来のOCRシステムでは印字と手書きで別々のモデルを用意し、切り替えが必要でした。

しかしYomiTokuは**単一のモデルで印字と手書きの両方を同時に認識**。これにより、印刷されたフォームに手書きで記入された申請書や、印刷物に手書きのメモが追加された資料など、現実の文書を自然に処理できます。

### 実務での成功事例

- **契約書の署名認識**：印刷された契約書の手書き署名を正確に認識
- **検査記録の処理**：印刷フォーマットに手書きで記入されたデータを一括処理
- **アンケートの自由記述**：選択肢と手書きコメントを同時に構造化

## 検索可能PDF生成とMCPサーバー対応

### 見た目と機能性を両立する検索可能PDF

v0.9.3で追加された**検索可能PDF生成機能**は、元の文書の見た目を完全に保ちながら、テキストの選択、検索、コピーを可能にします。

```bash

```

- **複数ページの結合**
- **ヘッダー・フッターの除外**（`--ignore_meta`）

### AI時代のワークフロー統合：MCPサーバー対応

v0.9.0で実装された**Model Context Protocol（MCP）サーバー対応**により、Claude DesktopなどのAIアシスタントから直接OCR機能を呼び出せるようになりました。


#### 📋 **ステップ1：Google Colabを開く**

1. [Google Colab](https://colab.research.google.com/)にアクセス
2. 「新しいノートブック」をクリック

#### 📋 **ステップ2：魔法のコードをコピペ**

```python
# =========================
# ① ライブラリをインストール
# =========================
!pip install -q yomitoku           # 日本語特化OCR
!apt-get -yqq install poppler-utils # PDF ⇄ 画像変換

# =========================
# ② PDF をアップロード
# =========================
from google.colab import files
uploaded = files.upload()          # 画面でファイルを選ぶだけ

# =========================
# ③ YomiToku で一括 OCR
# =========================
import os, subprocess, shlex, zipfile, pathlib, uuid
from tqdm.auto import tqdm         # 進捗バー表示用

out_root = f"ocr_results_{uuid.uuid4().hex[:8]}"
os.makedirs(out_root, exist_ok=True)

for fn in tqdm(uploaded.keys(), desc="OCRing"):
    in_path = f"/content/{fn}"
    if not in_path.lower().endswith(".pdf"):
        print(f"⚠️  Skipping non-PDF: {fn}")
        continue

    safe = pathlib.Path(fn).stem
    outdir = os.path.join(out_root, safe)
    os.makedirs(outdir, exist_ok=True)

    for fmt in ("html", "csv"):    # 読む用＆加工用
        cmd = (
            f'yomitoku {shlex.quote(in_path)} '
            f'-f {fmt} -o {shlex.quote(outdir)} '
            f'-v --figure --encoding utf-8-sig --combine'
        )
        subprocess.run(cmd, shell=True, check=True)

# =========================
# ④ ZIP に固めてダウンロード
# =========================
zip_name = f"{out_root}.zip"
with zipfile.ZipFile(zip_name, "w", zipfile.ZIP_DEFLATED) as zf:
    for root, _, files_in_dir in os.walk(out_root):
        for file in files_in_dir:
            zf.write(os.path.join(root, file))

print(f"📦  OCR 結果を {zip_name} にまとめました")
files.download(zip_name)
```

#### 📋 **ステップ3-5：実行から完了まで**

1. **PDFアップロード**：ファイル選択画面でPDFを選択
2. **処理待機**：進捗バーが表示され、自動実行
3. **結果ダウンロード**：ZIPファイルが自動ダウンロード
