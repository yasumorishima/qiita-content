# Google Apps Script Web Appでページ遷移できない問題と解決方法

![GAS_画面遷移.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/2433285/31effe06-b725-42e0-8bb0-c33dc4ecb786.png)


## はじめに

Google Apps Script (GAS) で複数ページを持つWeb Appを作成していると、こんな問題に遭遇しませんか？

- ボタンをクリックしても何も起こらない
- URLが変わらない
- ページが真っ白になる

私も日記アプリを作っている時に、何度もこの問題でハマりました。今回はその原因と解決方法を共有します。

## 問題の症状

以下のような普通のHTMLリンクやJavaScriptコードが動かない：

```html
<!-- これが動かない -->
<a href="?page=write">日記を書く</a>
```

```javascript
// これも動かない
function navigateTo(page) {
  window.location.href = '?page=' + page;
}
```

ボタンをクリックしても、URLが変わらず、ページも遷移しません。

## 原因：GAS Web Appはiframe内で動作している

実は、**GAS Web Appは常にiframe内で実行されています**。

これが問題の本質です。

### 何が起きているのか

1. ブラウザのアドレスバーには本番URL（`https://script.google.com/macros/s/.../exec`）が表示される
2. しかし実際のコンテンツはiframe内で動作している
3. iframe内のURLと、ブラウザのアドレスバーのURLは別物

そのため、`window.location.href`でURLを変更しても：
- iframe内のURLは変わる
- でもブラウザのアドレスバー（トップウィンドウ）のURLは変わらない
- GASの`doGet(e)`関数が呼ばれない
- 結果、ページ遷移が発生しない

### 確認方法

ブラウザのコンソール（F12キー）で確認できます：

```javascript
console.log('iframe内?:', window.self !== window.top);  // → true
console.log('iframe内URL:', window.location.href);
// → https://n-dotzx35...googleusercontent.com/userCodeAppPanel?createOAuthDialog=true

console.log('ブラウザURL:', window.top.location.href);
// → https://script.google.com/macros/s/.../exec
```

## 解決方法

### ステップ1：iframe内かチェックする

```javascript
const isInIframe = window.self !== window.top;
```

### ステップ2：トップウィンドウのURLを変更する

`window.location.href`ではなく、**`window.top.location.href`を使います**。

```javascript
function navigateTo(page) {
  // iframe内かどうかチェック
  const isInIframe = window.self !== window.top;

  // 本番URLを構築（あなたのDeployment IDに置き換える）
  const baseUrl = 'https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec';
  const newUrl = baseUrl + '?page=' + page;

  // iframe内ならトップウィンドウを変更
  if (isInIframe) {
    window.top.location.href = newUrl;  // ← これが重要！
  } else {
    window.location.href = newUrl;
  }
}
```

### ステップ3：HTMLを修正する

通常のリンクではなく、ボタン+JavaScript関数を使います。

```html
<!-- ❌ これは動かない -->
<a href="?page=write">日記を書く</a>

<!-- ✅ これが正しい -->
<button type="button" onclick="navigateTo('write')">
  日記を書く
</button>
```

**重要**: `type="button"`を指定しないと、フォーム送信として扱われる可能性があります。

## 完全なコード例

### HTML側

```html
<!DOCTYPE html>
<html>
<head>
  <title>日記アプリ</title>
</head>
<body>
  <h1>ホーム</h1>

  <button type="button" onclick="navigateTo('write')">
    日記を書く
  </button>

  <button type="button" onclick="navigateTo('past')">
    過去の日記
  </button>

  <script>
    function navigateTo(page) {
      const isInIframe = window.self !== window.top;
      const baseUrl = 'https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec';
      const newUrl = baseUrl + '?page=' + page;

      if (isInIframe) {
        window.top.location.href = newUrl;
      } else {
        window.location.href = newUrl;
      }
    }
  </script>
</body>
</html>
```

### GAS側（Code.gs）

```javascript
function doGet(e) {
  const page = e.parameter.page || 'home';

  switch(page) {
    case 'home':
      return HtmlService.createHtmlOutputFromFile('Home');
    case 'write':
      return HtmlService.createHtmlOutputFromFile('Write');
    case 'past':
      return HtmlService.createHtmlOutputFromFile('Past');
    default:
      return HtmlService.createHtmlOutputFromFile('Home');
  }
}
```

## デバッグのコツ

コンソールログを追加すると問題を特定しやすくなります：

```javascript
function navigateTo(page) {
  console.log('ページ遷移開始:', page);
  console.log('iframe内:', window.self !== window.top);
  console.log('現在のURL:', window.location.href);

  const isInIframe = window.self !== window.top;
  const baseUrl = 'https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec';
  const newUrl = baseUrl + '?page=' + page;

  console.log('遷移先URL:', newUrl);

  if (isInIframe) {
    window.top.location.href = newUrl;
  } else {
    window.location.href = newUrl;
  }
}
```

ブラウザのコンソール（F12キー）で、これらのログを確認できます。

## よくある間違い

### ❌ 間違い1：相対URLを使う

```javascript
// これは動かない場合がある
window.location.href = '?page=write';
```

iframe内URLが開発用URLの可能性があるため、正しいページに遷移しません。

### ❌ 間違い2：`window.location.href`だけ使う

```javascript
// これではiframe内URLしか変わらない
window.location.href = baseUrl + '?page=write';
```

`window.top.location.href`を使う必要があります。

### ❌ 間違い3：`<a>`タグをそのまま使う

```html
<!-- GAS Web Appではこれが動作しない -->
<a href="?page=write">日記を書く</a>
```

JavaScriptで明示的に遷移処理を書く必要があります。

## clasp使用時の注意

clasp（Google Apps Script CLI）を使っている場合：

1. `clasp push`でコードをアップロード
2. **GASエディタで手動デプロイが必要**
   - 「デプロイ」→「デプロイを管理」
   - 既存デプロイを編集
   - 「新しいバージョン」を選択
   - 「デプロイ」をクリック

`clasp push`だけでは、Web Appには反映されません。

## まとめ

- **GAS Web AppはiFrame内で動作する**（これを理解することが最重要）
- 通常の`window.location.href`ではなく **`window.top.location.href`を使う** 
- iframe検出: `window.self !== window.top`
- 本番URLをハードコードすると確実
- `<a>`タグではなく、`<button onclick="...">` + JavaScriptを使う
- `type="button"`を明示的に指定する

この問題は一度理解すれば簡単ですが、初めて遭遇すると原因が分かりにくいです。

同じ問題で困っている方の参考になれば幸いです！
