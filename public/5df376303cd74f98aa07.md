# Claude Codeにccstatuslineを導入して使用状況を可視化する方法

## はじめに

Claude Codeを使っていて「コンテキスト上限まであとどれくらい？」「このセッション、いくらかかってる？」と不安になったことはありませんか？

**ccstatusline**を使えば、こうした疑問を解消できます。ターミナル下部にモデル情報、トークン使用量、コンテキスト残量、セッションコストなどをリアルタイム表示する神ツールです。

この記事では、Claude Codeにccstatuslineをインストールして、見やすい監視機能を設定するまでの手順を、**コピペですぐ使える指示文**と共に解説します。

## ccstatuslineとは？

ccstatuslineは、Claude Code CLI用の高度にカスタマイズ可能なステータスラインフォーマッターです。

### 主な機能

- 🎨 **Powerlineスタイル対応** - 美しい矢印セパレーターとテーマ
- 📊 **リアルタイムメトリクス** - モデル名、トークン数、コストなど
- 🖥️ **インタラクティブTUI** - 直感的な設定画面
- 📏 **マルチライン対応** - 複数行の情報表示が可能

### GitHubでの評価

- ⭐ **2,000以上のスター**
- 👨‍💻 開発者：Matthew Breedlove
- 📦 リポジトリ：https://github.com/sirmalloc/ccstatusline

## なぜccstatuslineが必要なのか？

Claude Codeを使っていると、以下のような課題に直面します：

1. **コンテキスト上限が見えない** → いつ80%に達するか分からない
2. **セッションコストが不明** → 予算管理ができない
3. **モデルの確認が面倒** → どのモデルで作業中か忘れる
4. **5時間ブロックの管理** → リセットタイミングが分からない

ccstatuslineはこれらを**すべて可視化**してくれます。

## 🚀 クイックスタート（5分で完了）

以下の指示をClaude Codeにコピペするだけで、自動セットアップが完了します。

```
ccstatuslineをインストールして、見やすい監視機能を設定してください。

以下の手順で実行してください：

1. ~/.claude/settings.json を確認して、現在の設定を表示
2. ccstatuslineの設定を追加（bunx ccstatusline@latestを使用）
3. 以下のウィジェットを含む推奨設定を提案：
   - モデル名
   - コンテキスト使用率（使用可能率）
   - セッションコスト
   - ブロックタイマー
   - 入力/出力トークン数
   - Gitブランチ

4. 設定を保存してClaude Codeの再起動方法を教えてください
```

**たったこれだけ！** Claude Codeが自動的にインストールと設定を行います。

## 📝 ステップバイステップガイド

詳しく知りたい方向けに、各ステップを解説します。

### ステップ1：基本インストール

#### Claude Codeへの指示

```
ccstatuslineをClaude Codeにインストールしてください。

1. ~/.claude/settings.json を開いて現在の設定を確認
2. statusLine設定を追加（まだなければ）：
   - type: "command"
   - command: "bunx ccstatusline@latest"
   - padding: 0
3. 設定ファイルを保存
4. 変更内容を表示して、再起動が必要なことを教えてください
```

#### 設定ファイルの例

```json
{
  "statusLine": {
    "type": "command",
    "command": "bunx ccstatusline@latest",
    "padding": 0
  }
}
```

### ステップ2：TUIでのカスタマイズ

手動でカスタマイズしたい場合は、以下の指示を使います。

```
ccstatuslineのカスタマイズを手伝ってください。

1. ターミナルで `npx ccstatusline@latest` を実行する方法を教えてください
2. TUIでの主な操作方法を説明：
   - ウィジェットの追加/削除
   - 色のカスタマイズ
   - Powerlineモードの有効化
   - 設定の保存とインストール
3. 推奨するウィジェット構成を提案してください
```

#### TUIの操作方法

| キー | 機能 |
|------|------|
| ↑↓ | ウィジェット選択 |
| Enter | ウィジェット追加 |
| Delete/Backspace | ウィジェット削除 |
| c | 色の変更 |
| p | Powerline設定 |
| a | 整列 |
| i | インストール |
| t | タイムアウト設定 |
| l | 使用率/残り率トグル |

### ステップ3：推奨カスタム設定

監視に最適な構成をClaude Codeに依頼します。

```
以下のウィジェット構成でccstatuslineをセットアップしてください：

【1行目：重要な監視情報】
- Model Name（青色）
- Context Percentage (usable)（緑→黄→赤でグラデーション）
- Session Cost（黄色）
- Block Timer（紫色）

【2行目：詳細情報】
- Tokens Input（シアン）
- Tokens Output（マゼンタ）
- Tokens Total（白）
- Git Branch（緑色）

設定のポイント：
1. Powerlineモードを有効化
2. 256色モードまたはTruecolorを使用
3. 'a'キーで整列して見やすく調整
4. プレビューで確認してからインストール

実際の手順を教えてください。
```

## 🎯 推奨ウィジェット一覧

### 必須ウィジェット（監視用）

| ウィジェット | 説明 | 推奨色 |
|------------|------|--------|
| Model Name | 現在のモデル名 | 青 |
| Context % (usable) | 80%基準の使用率 | 緑→黄→赤 |
| Session Cost | セッションコスト | 黄 |
| Block Timer | 5時間ブロックタイマー | 紫 |

### オプション（詳細情報）

| ウィジェット | 説明 | 推奨色 |
|------------|------|--------|
| Tokens Input | 入力トークン数 | シアン |
| Tokens Output | 出力トークン数 | マゼンタ |
| Tokens Total | 合計トークン数 | 白 |
| Git Branch | 現在のブランチ | 緑 |
| Session Clock | セッション時間 | グレー |
| Working Directory | 作業ディレクトリ | 白 |

## 🔧 高度なカスタマイズ

### カスタムコマンドの追加

```
ccstatuslineにカスタムコマンドを追加してください：

1. 現在時刻を表示するウィジェット：
   コマンド: date +%H:%M
   
2. ccusageとの連携（使用量追跡）：
   コマンド: npx -y ccusage@latest statusline
   
3. 現在のコミットハッシュ：
   コマンド: git rev-parse --short HEAD

これらのCustom Commandウィジェットを追加する方法と、
タイムアウト設定の調整方法を教えてください。
```

### Powerlineフォントのインストール

文字化けを防ぐため、Nerd Fontのインストールが推奨されます。

**Windows:**
```bash
winget install DEVCOM.JetBrainsMonoNerdFont
```

**Mac:**
```bash
brew tap homebrew/cask-fonts
brew install --cask font-jetbrains-mono-nerd-font
```

**Linux:**
```bash
# 手動インストール
mkdir -p ~/.local/share/fonts
cd ~/.local/share/fonts
wget https://github.com/ryanoasis/nerd-fonts/releases/download/v3.0.0/JetBrainsMono.zip
unzip JetBrainsMono.zip
fc-cache -fv
```

## 🐛 トラブルシューティング

問題が発生した場合は、以下の指示をClaude Codeに送ります。

```
ccstatuslineのトラブルシューティングを手伝ってください：

1. 現在の設定を確認（~/.claude/settings.json）
2. ccstatuslineが正しく動作しているか確認
3. 以下の問題があれば解決策を提示：
   - ステータスラインが表示されない
   - 文字化けが発生している
   - 一部のウィジェットが動作しない
   - 設定変更が反映されない

4. Powerlineフォントのインストール状況も確認してください
```

### よくある問題と解決策

#### ステータスラインが表示されない

- Claude Codeを完全に再起動
- `~/.claude/settings.json`のJSON構文を確認
- `bunx --version`でBunが利用可能か確認

#### 文字化けが発生する

- Nerd Font（JetBrains Mono、Fira Codeなど）をインストール
- ターミナルのフォント設定を変更
- PowerlineモードをOFFにしてテスト

#### ウィジェットが動作しない

- Claude Code v1.0.85以降か確認（Session Cost用）
- Git情報は.gitディレクトリがある場所で動作
- カスタムコマンドはタイムアウトを調整（'t'キー）

## 💾 設定の保存と共有

設定をバックアップして、他の環境でも使えるようにします。

```
現在のccstatusline設定をバックアップしてください：

1. ~/.claude/settings.json の内容を表示
2. ccstatusline関連の設定を別ファイルに保存
3. 他のマシンで同じ設定を使う方法を教えてください
4. 設定のリセット方法も説明してください
```

```bash

cd ~

git init

git commit -m "Add Claude Code settings"
```
