## やったこと

### 1. ts-node → tsx に変更

テンプレートの音声生成スクリプトは `ts-node` で実行する設計だったが、ESMの関係で動かなかった。`tsx` に差し替えて解決。

```json
"voices": "npx tsx scripts/generate-voices.ts"
```

### 2. WAV長さ取得をPythonなしに

元の実装はPython3の`wave`モジュールを使っていた。Pythonを入れたくなかったので、WAVヘッダーを直接読むJavaScript実装に書き換えた。

### 3. 音声パラメータの調整

ここが一番試行錯誤した部分。VOICEVOXのAPIに渡すパラメータで音声の品質がかなり変わる。

```typescript
query.prePhonemeLength = 0;    // 冒頭の無音なし
query.postPhonemeLength = 0.1; // 末尾は少し余韻を残す
query.speedScale = 1.0;        // 通常速度
```

最初はテンプレートのデフォルト（1.2倍速）で使っていたが、冒頭の音声が小さくなる問題があったので1.0に戻した。

#### 試して失敗したこと

- **冒頭に「、っ、」を追加** → 口パク（音なし）の原因になった
- **`postPhonemeLength = 0`** → セリフの末尾がバッサリ切れて不自然
- **ffmpegの`stop_periods`で末尾無音削除** → 音声途中の無音（句読点の間など）まで削除されてめちゃくちゃになった

結局、VOICEVOX側のパラメータで制御するのが一番シンプルだった。ffmpegで後処理するのは小手先の対応で、根本的ではなかった。

### 4. durationInFrames自動更新

音声を生成した後、各セリフの再生時間（フレーム数）を`script.ts`に手動で書き戻す必要があった。これを自動化した。

ポイントは正規表現のマッチ方法。最初はセリフの`id`番号でマッチさせていたが、`id: 1`のパターンが`id: 10`〜`id: 19`にも当たってしまい、最初のセリフだけdurationがおかしくなるバグがあった。voiceFile名（`01_zundamon.wav`等）でマッチさせることで解決。

あと、計算したフレーム数に+5フレームのバッファを足している。ぴったりだと末尾が切れることがあるため。

### 5. 再生速度の二重適用を修正

テンプレートでは音声生成時に`speedScale`で速度調整し、さらに再生時にも`playbackRate`で調整していた。二重に適用されるので、再生側の調整を削除した。

## ワークフロー

最終的にはこの流れで動画が作れる。

1. Claude Codeに「〇〇の解説動画を作って」と伝える
2. セリフが`src/data/script.ts`に生成される
3. VOICEVOXを起動して`npm run voices`で音声生成
4. `npm start`でプレビュー確認（http://localhost:3000）
5. 修正があればClaude Codeに指示
6. `npm run build`で動画出力（out/video.mp4）

セリフの作成・修正はClaude Codeがやってくれるので、自分がやるのはコマンド実行とプレビュー確認だけ。
