# Claude Codeの使い方 - 実際の開発例

![クロードコード_1枚.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/2433285/e78f8bd9-b665-40c3-a00b-4cc12dd8f26e.png)


## はじめに

Claude CodeというAIアシスタントを使って、GAS（Google Apps Script）で日記アプリを開発しました。

この記事では、実際にどう使ったかを紹介します。

---

## Claude Codeとは

VSCode（エディタ）の中で動くAIアシスタントです。

普通のChatGPTと違って：
- ファイルを直接編集できる
- 複数ファイルを見て判断できる
- エラー箇所を直接修正できる

---

## 実際の使い方

### 例1：コード生成を依頼する

**私の指示**：
```
スプレッドシートからデータを取得する関数を書いて。
A列：日付
B列：内容
C列：気分
で取得してください。
```

**Claude Code**：
```javascript
function getAllData() {
  const ss = SpreadsheetApp.openById('YOUR_SPREADSHEET_ID');
  const sheet = ss.getSheetByName('Sheet1');
  const data = sheet.getDataRange().getValues();
  return data;
}
```

コードが生成されます。

---

### 例2：エラーを報告する

**私の指示**：
```
このコードがエラーになります：

ReferenceError: navigateTo is not defined
    at Write.html:123

該当コード：
<button onclick="navigateTo('home')">ホームへ</button>

どうすればいいですか？
```

**Claude Code**：
```
navigateTo関数が定義されていません。
以下の関数を<script>タグ内に追加してください：

function navigateTo(page) {
  const baseUrl = 'YOUR_APP_URL';
  window.top.location.href = baseUrl + '?page=' + page;
}
```

原因と解決策を教えてくれます。

---

### 例3：段階的に機能追加

**Phase 1**：
```
「まず、日記を保存する基本機能を作って」
```
→ Code.gsとHTMLが生成される

**Phase 2**：
```
「動きました。次に、過去の日記一覧を表示する機能を追加して」
```
→ 一覧表示のコードが追加される

**Phase 3**：
```
「一覧ができました。各エントリーに編集ボタンをつけて」
```
→ 編集ボタンと遷移処理が追加される

このように、一つずつ確認しながら進められます。

---

### 例4：問題を一緒に解決

**私**：
```
ページ遷移ができません。
リンクをクリックしてもURLが変わりません。

現在のコード：
<a href="?page=write">日記を書く</a>
```

**Claude Code**：
```
GAS Web AppはiFrame内で動作するため、
通常のhrefでは遷移できません。

以下のように変更してください：

1. navigateTo関数を追加
2. <a>タグを<button>に変更
3. window.top.location.hrefを使用

修正後：
<button type="button" onclick="navigateTo('write')">
  日記を書く
</button>
```

**私**：
```
ありがとうございます。でもまだ動きません。
```

**Claude Code**：
```
URLをハードコードする必要があります。
以下のように修正してください：

const baseUrl = 'https://script.google.com/macros/s/.../exec';
window.top.location.href = baseUrl + '?page=' + page;
```

このように、何度もやり取りしながら解決できます。

## 指示の出し方のコツ

実際に開発して学んだコツです。

### 1. 具体的に伝える

**悪い例**：
```
「アプリを作って」
```

**良い例**：
```
「日記アプリを作りたい。

データ：
- 日付
- 内容
- 気分（絵文字）

機能：
- 日記を保存
- 一覧表示
- 編集

保存先：Googleスプレッドシート

まず保存機能から作ってください」
```

何を、どう、どこに、を明確にします。

---

### 2. 段階的に進める

一度に「全機能を作って」と言わない。

1. 保存機能を作る → 動作確認
2. 一覧機能を追加 → 動作確認
3. 編集機能を追加 → 動作確認

このように、一つずつ確認しながら進めます。

最初に「全部一度に」と指示して、混乱した経験があります。

---

### 3. エラーは全文コピペ

省略せず、全部貼り付けます。

**悪い例**：
```
「エラーが出ました」
```

**良い例**：
```
「こんなエラーが出ました：

TypeError: Cannot read property 'value' of null
    at Form.html:42

該当行：
const name = document.getElementById('userName').value;

何が問題ですか？」
```

エラーメッセージ、行番号、該当コードを全部貼ります。

---

### 4. 期待する動作を伝える

「動かない」だけじゃなく、どう動いてほしいかも伝えます。

**悪い例**：
```
「保存ボタンが動きません」
```

**良い例**：
```
「保存ボタンをクリックしたら：

期待：
1. スプレッドシートにデータを保存
2. 画面上部に'保存しました'と表示
3. 3秒後にメッセージを消す

現状：
- クリックしても何も起きない
- コンソールエラーなし
```

---

### 5. わからないことは聞く

生成されたコードがわからないときは、説明を求めます。

**私**：
```
「このコードの動作を説明して：

google.script.run
  .withSuccessHandler(onSuccess)
  .withFailureHandler(onError)
  .saveData(data);
```

**Claude Code**：
```
このコードは以下の処理をしています：

1. google.script.run: HTMLからGAS関数を呼び出す
2. withSuccessHandler: 成功時の処理
3. withFailureHandler: 失敗時の処理
4. saveData(data): サーバーサイド関数を呼び出し

重要：非同期処理なので、結果はコールバックで受け取る
```

理解しながら進められます。

## 実際に遭遇した問題

### 問題1：ページ遷移ができない

**症状**：
```html
<a href="?page=write">日記を書く</a>
```
クリックしてもページが変わらない。

**原因**：
GAS Web AppはiFrame内で動作するため、通常のhrefでは遷移できない。

**解決策**：
Claude Codeに質問して、`window.top.location.href`を使う方法を教えてもらった。

---

### 問題2：絵文字でエラー

**症状**：
```javascript
console.log('🚀 アプリ起動');
```
→ `Uncaught SyntaxError: Invalid or unexpected token`

**原因**：
JavaScriptコード内の絵文字がGASのレンダリングでエラーになる。

**解決策**：
Claude Codeに報告して、絵文字を削除する方法を教えてもらった。
```javascript
console.log('[DEBUG] アプリ起動');
```

---

### 問題3：テンプレート変数が表示されない

**症状**：
```html
<h1><?= userName ?></h1>
```
→ 空白が表示される

**原因**：
Code.gsで変数をセットし忘れていた。

**解決策**：
Claude Codeに聞いて、以下を追加：
```javascript
function doGet(e) {
  const template = HtmlService.createTemplateFromFile('Page');
  template.userName = 'たろう';  // これを追加
  return template.evaluate();
}
```

## 良かった点

### 1. エラー修正が楽

エラーメッセージをコピペすると、原因と解決策が返ってきます。

StackOverflowで検索する時間が減りました。

### 2. 学びながら開発できる

「なぜこう書くのか？」と質問すると、説明してくれます。

コードを書くだけでなく、理解も深まります。

### 3. 複数ファイル管理が楽

11個のHTMLファイルを管理していましたが、整合性を保つのが簡単でした。

変数名の変更も一括で対応してくれます。

### 4. GASへのアップロードが楽

clasp CLIを使うと、ローカルで編集したファイルをGASに一括アップロードできます。

```bash
clasp push
```

これだけで全ファイルがGASにアップロードされます。

従来は：
- GASエディタを開く
- ファイルを一つずつ開く
- コードをコピペ
- 保存
- 次のファイル...（12回繰り返し）

clasp使用後：
- VSCodeで編集
- `clasp push` 一発
- 完了

12ファイルの手動コピペ作業がなくなりました。

## 注意点

### 1. 生成されたコードは必ず確認

たまに間違いがあります。特に：
- ファイル名のtypo
- 関数名の不一致
- ロジックの勘違い

動かして確認する習慣が大事です。

### 2. GAS特有の仕様で詰まることも

例：
- ページ遷移がiframe内で動かない
- テンプレート変数の扱い
- google.script.runの非同期処理

エラーを報告すれば、修正してくれます。

### 3. 一度に全部作ろうとしない

最初は「全機能を一度に」と指示してしまいました。

結果、混乱しました。

解決策：1機能ずつ、段階的に実装

実際の使い方：

2. 段階的に進める
3. エラーは全文コピペ
4. 期待する動作を伝える
5. わからないことは聞く

注意点：
- 生成されたコードは必ず確認
- 一度に全部作ろうとしない
- エラーが出たら報告する

対話しながら開発できるので、初心者でも複雑なアプリが作れます。
